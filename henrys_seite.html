Nesting

The SCSS code will follow the same visiual hierarchy like the html. If a code for one element is written inside another element it will behave like :
Outer Element (Parent) > Inner Elemtent (Child)

Example:

//CSS:

.content {
     font-size: 25px;
     color: green;
}

.content p {
     color: red;
}
>
//SCSS:

.content {
     font-size: 25px;
     color: green;

            p {
                  color: red;
               }
}

Nested Properties

You can also nest proberties which share the same "forename". Therefore you don't need to write repeatedly the identical proberty over and over again.

Example:

//CSS:

.class {
     font-family: Helvetica;
     font-size: 18px;
     font-weight: bold;
}
>
//SCSS:

.class {
     font: {
          family: Helvetica;
          size: 18px;
          weight: bold;
      }
}

Referencing Parents Selectors: &

Inside a rule you don't need to write out the parent selector again. The & is replacing the parent selector within the rule. The & must be written before a compound selector, but a suffix can be followed that will be added to the parent selector.

Example:

//CSS

.class {
     background-color: red; }

.class a {
     text-decoration: none; }

.class a:hover {
     color: blue; }
>
//SCSS

.class {
     background-color: red;
          a {
               text-decoration: none;
                    &:hover {color: blue;}
               }
}
