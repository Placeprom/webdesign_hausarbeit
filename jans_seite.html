Mixins

A lot of work in CSS requires the copying & pasting of styles. This makes most of the work repetetive and tedious to write.
This is especially evident with vendor prefixes in CSS3, because you have to take every unique property of a browsers rendering
engine into account. Mixins made this a lot easier.

What are Mixins? 

Mixins let you group together certain declarations in CSS, so that you will be able to use them, instead of re-typing/copying the
exact same declarations all over you websites. 

It is even possible to include values in these mixin, giving you more control and flexibillity. 
 

Here's an example for transform.

@mixin transform(&property) {
-webkit-transform: $property;
-ms-transform: $property;
transform: $property;
}


.box { @include transform(rotate(30deg)); }

To create a mixin you use the @mixin directive and give it a name. We've named our mixin transform. We're also 
using the variable $property inside the parentheses so we can pass in a transform of whatever we want. After 
you create your mixin, you can then use it as a CSS declaration starting with @include followed by the name of the mixin.
        

:root {
    --heading-style: {
        text-transform: uppercase;
        color: var(--rot);
        border-bottom: 2px solid var(--rot);
    };
}         
                


                
h1 {
    @apply --heading-style;
}
